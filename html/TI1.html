Potęgowanie i pierwiastkowanie w PHP

Wstęp

Potęgowanie i pierwiastkowanie to jedne z podstawowych działań matematycznych. Język PHP posiada wbudowane funkcje i operator umożliwiające wykonywanie tych działań, są to:

  • Funkcja pow()
  • Operator **
  • Funckja sqrt()

Poniżej zostanie omówiony każdy z powyższych elementów.

Funkcja pow()

Funkcja pow() jest podstawową funkcją używaną przy działaniach potęgowania i pierwiastkowania. Przyjmuje dwa parametry, kolejno podstawę i wykładnik.

Składnia

pow(<podstawa>,<wykładnik>);

Przykład 1 - Potęgowanie

KOD

<?php
 echo pow(2, 0);
 echo "<br>";
 echo pow(2, 1);
 echo "<br>";
 echo pow(2, 2);
 echo "<br>";
 echo pow(2, 3);
 echo "<br>";
 echo pow(2, 4);
?>

REZULTAT

1
2
4
8
16

Przykład 2 - Pierwiastkowanie

KOD

<?php
 echo pow(1, 1/2);
 echo "<br>";
 echo pow(8, 1/3);
 echo "<br>";
 echo pow(81, 1/4);
 echo "<br>";
 echo pow(1024, 1/5);
 echo "<br>";
 echo pow(15625, 1/6);
?>

REZULTAT

1
2
3
4
5


Operator **

W wersji języka PHP 5.6 wprowadzono operator potęgowania. Używa się go w taki sam sposób jak inne podstawowe znaki matematyczne takie jak plus i minus.

Składnia

<podstawa>**<wykładnik>;

Przykład 1 - Potęgowanie

KOD

<?php
 echo 2**0;
 echo "<br>";
 echo 2**1;
 echo "<br>";
 echo 2**2;
 echo "<br>";
 echo 2**3;
 echo "<br>";
 echo 2**4;
?>

16

<?php

 echo "<br>";
 echo 8**(1/3);
 echo "<br>";

 echo "<br>";
 echo 1024**(1/5);

KOD

 echo "<br>";

Wyjątki, throw, try i catch w PHP

Wyjątki

W PHP, wyjątki pozwalają na eliminację błędów które uniemożliwiają dalsze funkcjonowanie aplikacji. W tym celu wykorzystuje się trzy funckje wyjątków:

  • throw
  • try
  • catch

Ponieżej zostanie omówiony każdy z powyższych elementów.

Throw

Funkcja throw pozwala określić jaki wyjątek ma zostać "wyrzucony" w przypadku wystąpienia błędu. Nadaje się mu określoną wiadomość oraz kod który jest następnie wyświetlany użytkownikowi.

Składnia

function <nazwa funkcji>(<parametry>){
 if(<warunek wystąpienia błędu>) {
  throw new Exception("<wiadomość błędu>",<kod błędu>);
 }
  return <wartość którą funkcja zwraca bez wystąpienia błędu>;
 }

Przykład 1

Aby zaprezentować przykład skonstruowałem funkcję "sprawdzam", która w przypadku gdy parametr "$numer" jest większy od 10 zwróci wyjątek z wiadomością "Wprowadzona wartość jest większa od 10" oraz kodem błędu 5.

KOD

<?php
 function sprawdzam($numer) {
  if($numer>10) {
 throw new Exception("Wprowadzona wartość jest większa od 10.",5);
 }
  return true;
 }

 sprawdzam(11);

 echo "Przez błąd, ta wiadomość nie zostanie wyświetlona.";
?>

REZULTAT

Fatal error: Uncaught Exception: Wprowadzona wartość jest większa od 10. in /home/s48230/public_html/Semestr4/Technologie Internetowe/Prez1/P6.php:11 Stack trace: #0 /home/s48230/public_html/Semestr4/Technologie Internetowe/Prez1/P6.php(16): sprawdzam() #1 {main} thrown in /home/s48230/public_html/Semestr4/Technologie Internetowe/Prez1/P6.php on line 11

Jak widać, wprowadzona wartość 11 dla funkcji "sprwadzam" wyrzuciła wyjątek z przygotowaną przez nas wiadomością. Aplikacja przestała jednak działać i nie uruchomiła reszty kodu, aby temu zapobiec wykorzystamy funkcje try oraz catch.


Try i catch

Gdy już umiemy przygotować wiadomość oraz kod dla występującego wyjątku, kolejnym krokiem będzie przygotowanie aby aplikacja nie przestawała działać podczas jego wystąpienia. Do tego posłuży try, w którym umieszczone zostaną funkcje które podejrzewamy o wystąpienie wyjątki, a aplikacji po ich wykonaniu nie zatrzyma się.

Nierozłącznym elementem try jest catch, które pozwala wychwycić wyjątek który występił w obrębie try. Dzięki niemu będziemy mogli określić co aplikacja ma zrobić w przypadku wystąpienia wyjątku.

Składnia try

try {

<podejrzane funkcje>;

}

Składnia catch

catch(Exception $e) {

<funkcje które zostaną wykonane po wystąpieniu wyjątku>;

}

Przykład 1

KOD

<?php
 function sprawdzam($numer) {
  if($numer>10) {
 throw new Exception("Wprowadzona wartość jest większa od 10.",5);
 }
  return true;
 }

 try {
  sprawdzam(11);
  echo "Nie wystąpił żaden błąd";
 }

 catch(Exception $e) {
  echo "Wiadomość błędu: ".$e->getMessage();
  echo "<br>";
  echo "Kod błędu: ".$e->getCode();
 }
?>

REZULTAT

Wiadomość błędu: Wprowadzona wartość jest większa od 10.
Kod błędu: 5

W obrębie funkcji try została umieszczona wcześniej przygotowana przez nas funkcja zwracająca wyjątek, który następnie został wyłapany w obrębie funkcji catch w której została przygotowana wiadomość towarzysząca występującemu wyjątkowi. Warto zauważyć, że przez występujący wyjątek w obrębie funkcji try, funkcja echo nie została wywołana.

Aby przygotować formułkę przywoływaną przez catch, możemy posłużyć się wiadomością i kodem określonym przez nas w funkcji throw, do ich wywołania wykorzystujemy:

  • $e->getMessage() które zwraca określoną wiadomość błędu.
  • $e->getCode() które zwraca określony kod błędu.

Przykład 2

KOD

<?php
 function sprawdzam($numer) {
  if($numer>10) {
 throw new Exception("Wprowadzona wartość jest większa od 10.",5);
 }
  return true;
 }

 try {
  sprawdzam(9);
  echo "Nie wystąpił żaden błąd";
 }

 catch(Exception $e) {
  echo "Wiadomość błędu: ".$e->getMessage();
  echo "<br>";
  echo "Kod błędu: ".$e->getCode();
 }
?>

REZULTAT

Nie wystąpił żaden błąd

W tym przypadku try nie wyłapało błędu i wykonało wszystkie znajdujące się w jego obrębie funkcje.

Połączenie pierwiastkowania i wyjątków

Przykład 1

KOD

<?php
 function pierwiastekdrugiegostopnia($podstawa) {
  if($podstawa<0) {
 throw new Exception("Podstawa nie może być ujemna dla pierwiastka
 drugiego stopnia",123456789);
 }
  echo "Pierwiastek drugiego stopnia dla ".$podstawa." wynosi: ".$podstawa**(1/2)."<br>";
 }

 try {
  pierwiastekdrugiegostopnia(1);
 }

 catch(Exception $e) {
  echo "Wiadomość błędu: ".$e->getMessage();
  echo "<br>";
  echo "Kod błędu: ".$e->getCode();
 }

 try {
  pierwiastekdrugiegostopnia(-1);
 }

 catch(Exception $e) {
  echo "Wiadomość błędu: ".$e->getMessage();
  echo "<br>";
  echo "Kod błędu: ".$e->getCode();
 }
?>

REZULTAT

Pierwiastek drugiego stopnia dla 1 wynosi: 1
Wiadomość błędu: Podstawa nie może być ujemna dla pierwiastka drugiego stopnia
Kod błędu: 123456789

Przygotowałem funkcję która zwraca pierwiastek kwadratowy podanej wartości. Wprowadziłem w niej wyjątek który wyświetla się gdy podana wartość jest ujemna. W rezultacie można zaobserować efekt dla poprawnie i niepoprawnie wprowadzonych wartości.

Uwaga! Trzeba pamiętać, że po każdym wywołaniu funkcji try musi występować funkcja catch.
